# Лабораторная работа №2 — Коллекция объектов (Python 3.x)
## Управление автобусным парком (Вариант 5)

---

## Цель работы

- Научиться проектировать **классы-контейнеры** для агрегации объектов.
- Понять разницу между моделью сущности (`Bus`) и управляющей сущностью (`BusFleet`).
- Реализовать механизмы поиска, сортировки и фильтрации данных внутри коллекции.
- Освоить переопределение **магических методов** для работы с итерируемыми объектами (`__len__`, `__iter__`, `__getitem__`).

---

## Идея

Создать "умный" автобусный парк, который выступает единой точкой управления для множества автобусов:

- **Автоматизация**: Система не допускает появления двух автобусов с одинаковым `id`.
- **Безопасность**: Контейнер принимает только объекты класса `Bus`, защищая коллекцию от некорректных типов данных.
- **Гибкость**: Предоставляет методы для анализа данных (поиск по id и маршруту, сортировка по параметрам, а также фильтрация автобусов по состоянию).

## Реализованный класс-контейнер

### BusFleet

Класс, представляющий собой агрегатор (контейнер) для объектов из ЛР-1.

**Атрибуты экземпляра:**
- `_items` — **закрытый список** `[]` (инкапсулированное хранилище объектов `Bus`).

---

## Ключевые механизмы коллекции

### 1. Защита целостности данных
В методе `add(item)` реализована двойная проверка:
* **Type Checking**: Использование `isinstance()` гарантирует, что в список попадут только объекты `Bus`.
* **Unique Constraint**: Перед добавлением система проверяет `id`. Если автобус уже существует, выбрасывается `ValueError`.

### 2. Магические методы (Pythonic Interface)
Для того чтобы контейнер вел себя как стандартный список Python, переопределены:
* `__len__` — поддержка функции `len(fleet)`.
* `__iter__` — поддержка циклов `for bus in fleet:`.
* `__getitem__` — доступ к элементам по индексу (например, `fleet[0]`).

### 3. Бизнес-логика и аналитика
* `find_by_id(id)` — поиск автобуса по уникальному идентификатору.
* `find_by_route(route)` — поиск автобусов по маршруту.
* `sort_by_capacity()` — сортировка по вместимости.
* `sort_by_speed()` — сортировка по скорости.
* `get_active()` — метод возвращает **новый экземпляр** класса `BusFleet`, содержащий только автобусы со статусом "На маршруте".
* `get_in_park()` — возвращает автобусы со статусом "На парковке".
