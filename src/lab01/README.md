- Освоить объявление пользовательских классов в Python

## Функционал программы
Создан автопарк, состоящий из автобусов, которые могут:

- Посадка пассажиров — с проверкой свободных мест и статуса автобуса
- Высадка пассажиров — с обновлением статистики перевозок
- Выезд на маршрут — только при достаточном топливе и статусе "в парке"
- Завершение маршрута — с подсчетом пробега и расхода топлива
- Отправка в ремонт и завершение ремонта — изменение статуса
- Авария — имитация происшествия с пострадавшими пассажирами
- Запрет недопустимых действий — нельзя посадить пассажиров в автобус в ремонте или после аварии

Программа предоставляет удобный интерфейс через свойства (@property) и скрывает внутреннюю реализацию.

## Реализованный класс

### Bus

Главный класс, моделирующий автобус. Содержит всю основную логику работы с объектом автобуса.

**Атрибуты класса:**

- total_buses: всего создано автобусов
- total_passengers_transported: всего перевезено пассажиров всеми автобусами

Используются для глобальной статистики по всему автопарку.

---

**Защищенные поля (инкапсуляция):**

- `_route_number` — номер маршрута
- `_capacity` — вместимость автобуса
- `_speed` — средняя скорость
- `_driver_name` — имя водителя
- `_passenger_count` — текущее количество пассажиров
- `_status` — состояние автобуса (в парке, на маршруте, в ремонте, авария)
- `_mileage` — пробег
- `_fuel_level` — уровень топлива
- `_id` — уникальный идентификатор автобуса

Это инкапсуляция — данные защищены от прямого изменения. Доступ только через методы и свойства.

---

## Свойства @property

- `route_number` — получение номера маршрута (только чтение)
- `capacity` — получение вместимости (только чтение)
- `speed` — чтение и изменение скорости (с проверкой статуса)
- `driver_name` — чтение и изменение имени водителя
- `passenger_count` — получение текущего числа пассажиров (только чтение)
- `status` — получение статуса автобуса (только чтение)
- `fuel_level` — получение уровня топлива (только чтение)
- `mileage` — получение пробега (только чтение)
- `id` — получение ID автобуса (только чтение)

Свойства позволяют безопасно работать с данными и добавлять логику при доступе.

---

## Валидация (из модуля validate.py)

- `val_route_number()` — проверка номера маршрута (непустая строка)
- `val_capacity()` — проверка вместимости (положительное целое)
- `val_speed()` — проверка скорости (положительное число, не больше 120)
- `val_driver_name()` — проверка имени водителя (непустая строка, формат ФИО)
- `val_passenger_count()` — проверка посадки (статус, свободные места)

Все функции проверяют корректность данных и выбрасывают исключения при ошибках.

---

## Магические методы

- `__init__` — конструктор, создание объекта
- `__str__` — красивый вывод для пользователя (через print)
- `__repr__` — технический вывод для разработчика
- `__eq__` — сравнение объектов (по маршруту и водителю)
- `__lt__` — сравнение "меньше" (по вместимости, для сортировки)

Делают объекты удобными для работы, вывода и сравнения.

---

## Бизнес-методы

- `board_passengers(count)` — посадка пассажиров с проверкой мест и статуса
- `disembark_passengers(count)` — высадка пассажиров, обновление статистики
- `start_route()` — выезд на маршрут (проверка топлива и статуса)
- `finish_route()` — завершение маршрута (расчет пробега и топлива)
- `send_to_repair()` — отправка в ремонт
- `complete_repair()` — завершение ремонта (сброс параметров)
- `accident()` — имитация аварии (пострадавшие пассажиры)

Реализуют основную логику работы автобуса и изменяют его состояние.

---

## Статусы автобуса (Enum)

- `IN_PARK` — "в парке"
- `ON_ROUTE` — "на маршруте"
- `REPAIR` — "в ремонте"
- `ACCIDENT` — "авария"

Ограниченный набор состояний, через которые может проходить автобус.
